Voxel Sandbox Build Docs

Living documentation for game.html. It is self-contained and has no external assets.

Architecture Overview

Single-File Runtime

game.html contains all HTML, CSS, and JavaScript inline. The only network dependency is the required pinned Three.js module graph resolved through the mandatory import map.

World Model

The world is generated from hard-coded seed 1337. Blocks are queried procedurally from deterministic value noise, biome rules, cave carving, tree shapes, water fill, and a localStorage edit overlay.

Chunk Meshes

Chunks are 16 by 16 columns and 72 blocks tall. Each chunk builds independent opaque and water meshes, emits only visible faces, adds per-vertex ambient occlusion darkening, and streams within render distance 8.

Player and Interaction

The camera is first-person at eye height. Pointer-lock mouse-look, WASD movement, sprint, jump, gravity, AABB block collision, fly/no-clip, DDA raycast targeting, block break/place, hotbar, palette, and persistent edits are all handled inline.

Rendering and Atmosphere

A canvas-generated texture atlas supplies all block surfaces. Directional sun, ambient light, fog, transparent water, a CSS gradient sky, day-night lighting changes, drifting procedural clouds, synthesized Web Audio sounds, and a blocky wandering mob provide atmosphere.

Instrumentation

The F3 HUD reports FPS, coordinates, chunk coordinates, loaded/meshed chunk counts, render distance, draw calls, triangle count, seed, mode, current target, selected block, edit count, and the spawn-area block hash.

Key Technical Decisions

  • Use deterministic value-noise and hash functions instead of Math.random() for terrain, trees, caves, ores, and clouds so the default world remains reproducible.
  • Keep edits as an overlay map instead of mutating generated terrain. This preserves deterministic generation and lets localStorage save only changed blocks.
  • Mesh water separately with transparency and hidden-water-neighbor culling so shorelines remain readable while underwater terrain stays visible.
  • Use direct DDA voxel raycasting for precise break/place targeting and an independent wireframe cube for the crosshair highlight.
  • Throttle chunk building based on measured FPS so initial streaming fills the render distance without freezing normal play; nearby chunks are sorted first and low-FPS frames skip some mesh work.
  • Render at device pixel ratio 1 for stable desktop performance; the procedural block textures remain crisp because the atlas uses nearest-neighbor sampling.
