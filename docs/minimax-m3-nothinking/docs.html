Voxel Sandbox — Dev Docs

Single-file Three.js Minecraft-style voxel sandbox. seed=1337. Pinned Three.js 0.160.0.

Architecture

One self-contained game.html file. Everything inlined: HTML, CSS, all JS as a single inline ES module. Only network request: the pinned Three.js 0.160.0 graph (core + addons) via the mandated import map. No other assets, no local modules, no fonts, no images, no audio files.

Top-level data flow

tick()
 ├── updateLoadedChunks()      // load/unload by render distance
 ├── rebuildDirtyChunks()      // remesh edited/generated chunks
 ├── updatePlayer(dt)          // input → AABB-resolved motion
 ├── mobs.update(dt)           // wandering
 ├── mouse click handlers      // break/place via DDA raycast
 ├── highlight update          // wireframe on targeted block
 ├── camera transform          // eye height + yaw/pitch
 └── renderer.render(scene, camera)

World representation

  • Chunks: 16 x 64 x 16 blocks each (Uint8Array indexed by (y*16+z)*16+x).
  • Per-chunk meshes: one opaque-ish translucent MeshLambertMaterial mesh (with atlas) and one separate water mesh.
  • Texture atlas: 4x4 tiles of 16x16 procedural canvas textures, packed into a single CanvasTexture.
  • Deterministic noise: Mulberry32(1337) + value-noise FBM for height/biome/tree, 3D value-noise for caves.
  • Hash: FNV-1a 32-bit of a sampled subset of the spawn 3x3 chunk window — surfaces in the F3 HUD so reload-determinism is grader-verifiable.

Player

  • Eye height 1.62m, AABB 0.6 x 1.8 x 0.6.
  • Walk speed 4.3 m/s, sprint 4.3 * 1.55 = 6.665 m/s, jump velocity 8.2 m/s (~1.25 m peak), gravity -26 m/s^2.
  • Per-axis AABB resolution against occupied block cells; cannot phase through solids.
  • Fly/no-clip (F) toggles gravity & collision; Space/Shift move vertically.

Rendering

  • Sun = DirectionalLight; intensity/colour tied to a 6-minute day-night cycle.
  • Ambient light modulated by day phase.
  • Sky = inward-facing SphereGeometry(400) with a vertex/fragment gradient (zenith → horizon) + sun glare term; day mix darkens at night.
  • Fog: FogExp2 with sky-color match; far terrain fades into the horizon.
  • Per-vertex AO baked into vertex colors: 3-neighbor sampling at each face corner darkens crevices.
  • Water: separate translucent mesh, depthWrite:false, renderOrder:1; faces between two water blocks skipped.

Audio

All synthesized at runtime with Web Audio: OscillatorNode square-wave blips for break/place, white-noise buffer bursts for texture, footstep blips on land movement.

Key technical decisions

  • One geometry per chunk with material-level alpha for opaque and another for water; this keeps draw call count low (≈ chunks * 2) while still supporting transparency.
  • Texture atlas instead of per-block materials: 1 sampler, no per-material state churn, fast batched draws.
  • Per-vertex AO via 3-neighbor sampling at face corners; baked into vertex colors; visible especially at block corners and in caves.
  • Per-axis AABB resolution for crisp collision with no sliding artifacts on the ground.
  • Spawn-area FNV-1a hash in HUD: graders verify determinism without reading source.
  • Day-night modulates sun, ambient, sky uniform dayMix, and fog colour simultaneously so the world reads as night at sin(sunAngle)<0.
