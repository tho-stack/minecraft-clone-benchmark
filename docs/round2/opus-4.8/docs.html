Voxel Sandbox — Living Documentation

A single-file, browser-native Minecraft-style voxel game built on Three.js 0.160.0. Seed 1337.

Architecture Overview

game.html is one self-contained file. The only network request is the pinned Three.js module graph (declared via an importmap); everything else — textures, sounds, fonts, world data — is generated procedurally at runtime. It runs by opening the file directly over file://.

Rendering

Three.js WebGLRenderer, a 70° FOV perspective camera, a directional "sun" + ambient + hemisphere lighting, exponential-ish distance fog, and a gradient sky.

World

Seeded value/Perlin noise drives terrain height, biomes, caves and tree placement. The world is split into 16×16 columns (chunks), each independently meshed with hidden-face culling and baked ambient occlusion.

Streaming

Chunks within the render distance are generated/meshed on demand (throttled per frame) and freed when the player walks away — both observable via the HUD chunk count.

Player

First-person controller with pointer-lock mouse-look, AABB collision, gravity/jump, sprint, and a fly/no-clip mode. Raycast block break/place with a wireframe highlight.

Key Technical Decisions

  • Single file, pinned CDN. Three.js 0.160.0 is the only fetched resource, resolved through an import map that maps the bare three specifier (the unused three/addons/ entry was dropped since no addon is imported). All assets are procedural so a missing file is impossible.
  • Face-culled per-chunk meshes. Each chunk builds one BufferGeometry with only the faces exposed to air, plus a separate transparent mesh for water — keeping triangle counts far below cube-per-block.
  • Determinism. All terrain shape derives from a seeded PRNG/noise (no Math.random for shape), so the spawn-area hash in the HUD is identical on every reload.
